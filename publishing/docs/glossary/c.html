Container

Digital publications are made up of many different files: HTML documents, images, CSS files and scripts, among others. In order to simplify their distribution, they are all bundled together in a single container file.

EPUB, for example, uses ZIP for its container format. EPUB files can be opened by any ZIP tools, an operation that is made even easier by changing their extension from .epub to .zip (i.e., most computer operating systems have built-in tools to open files with .zip extensions).

Another common name for the EPUB container file is the "OCF file" or "OCF Container", as the requirements for zipping the content files together is defined in the EPUB Open Container Format (OCF) specification.

Content Provider

The organization directly responsible for providing EPUB content to readers, whether through an online bookstore, corporate Web site, library lending program or otherwise.

Core Media Type

A media type that all reading systems must support to ensure interoperability of content. For example, EPUB 3 requires reading systems be able to render XHTML5 and SVG documents, MP3 and MP4 audio, a variety of image formats, etc. Therefore, these are all core media types of EPUB 3.

If the media type of a resource is not core, it is considered a foreign resource.

CSS

The Cascading Style Sheet language is used to control the rendering of content on reading systems with a visual display screen, but does provide capabilities for rendering in other media such as braille and aural.

CSS Viewport

A screen on a reading system capable of visually rendering the EPUB using the defined CSS styles.
