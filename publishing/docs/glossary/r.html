Reader

The person who consumes an EPUB.

Not to be confused with a reading system, which is the application used to read a publication.

Reading System

Reading system is a term commonly used with EPUB to describe the application that a user uses to read their digital publication.

An EPUB reading system is a device capable of rendering an EPUB. A reading system may be a single dedicated device or it could be a combination of hardware and software and assistive technology. An iPad on its own is not a reading system, for example, but when iBooks is run on it the hardware and software constitute a reading system. Discussions about accessible reading systems are complicated by the wide variety of interplay that is possible between devices and software.

Remote Resource

A publication resource that is not included in the container. Audio and video resources are allowed to be housed on the Web because of their potential size. Resources referenced in package document link elements can also live outside the contianer (e.g., metadata records).

Rendition

A rendition of a publication is one presentation of its content.

All publications consist of one default presentation, but it is technically possible for more than one presentation to be bundled together. For example, a digital publication could have separate English and French versions, or a fixed layout and reflowable version for readers to choose from. EPUB 2 even allows alternative formats, such as a PDF version of the content, to be bundled inside the container.

A publication that, for example, has both English and French versions together in a single presentation is not a multiple rendition publication. Each rendition has to be unique — in EPUB, this means that each has its own package document.

In practice, however, renditions are more theoretical than practical at this time. A means of selecting between possible renditions was developed for EPUB 3, but has never gained any traction either with publishers or reading system developers.
